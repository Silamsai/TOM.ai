---
description: How to deploy the TOM.ai backend to AWS EC2
---

Follow these step-by-step instructions to deploy the TOM.ai backend to an AWS EC2 instance and connect it to your frontend.

### Step 1: Launch an AWS EC2 Instance
1. Go to your AWS Console, navigate to **EC2**, and click **Launch Instance**.
2. Select **Ubuntu Server 22.04 LTS (HVM)** as the Amazon Machine Image (AMI).
3. Choose an instance type (e.g., `t2.micro` or `t3.micro` for free tier, or `t3.small` for production).
4. Select or create a **Key Pair (.pem)** and download it to your computer (e.g., `tom-key.pem`).
5. Under **Network settings**, configure a **Security Group**:
   * Add a rule to allow **SSH** (Port 22) from your IP.
   * Add a Custom TCP rule to allow **Port 5000** (TOM.ai backend endpoint) from anywhere (`0.0.0.0/0`).
6. Click **Launch Instance** and copy its **Public IPv4 Address**.

---

### Step 2: Connect to the EC2 Server
Open your local terminal (Powershell or Bash) where your `tom-key.pem` key is saved:

1. Restrict permissions of your private key (required on Mac/Linux, recommended on Windows):
   ```bash
   # On Mac/Linux:
   chmod 400 tom-key.pem
   ```
2. Connect to your EC2 instance via SSH:
   ```bash
   ssh -i tom-key.pem ubuntu@<YOUR_EC2_PUBLIC_IP>
   ```

---

### Step 3: Install Node.js and Git
Once connected to the remote EC2 command prompt, install the runtime dependencies:

```bash
# Update local packages List
sudo apt-get update

# Install Node.js v20.x environment
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs git build-essential
```

Verify the installation:
```bash
node -v   # Should show v20.x.x
npm -v    # Should show v10.x.x
```

---

### Step 4: Clone Repository and Setup Backend
1. Clone your project repository onto the EC2 server:
   ```bash
   git clone <YOUR_GIT_REPOSITORY_URL>
   cd TOM.ai/backend
   ```
2. Install Node production dependencies:
   ```bash
   npm install --production
   ```

---

### Step 5: Configure Backend Environment
Create a `.env` file containing the environment configuration variables:
```bash
nano .env
```
Copy and paste your configurations (adjusting MongoDB, AI keys, and other credentials) inside the text editor:
```env
PORT=5000
MONGO_URI=mongodb+srv://...  # Your database URL (e.g., MongoDB Atlas connection)
JWT_SECRET=your_jwt_secret_key

# Generative AI Config
GEMINI_API_KEY=your_gemini_key

# Email Sending Providers (Resend/SendGrid/Gmail)
GMAIL_USER=...
GMAIL_PASS=...
RESEND_API_KEY=...
```
*   Save the file by pressing `Ctrl + O`, hitting `Enter`, and exit with `Ctrl + X`.

---

### Step 6: Keep the Backend Running with PM2
Install PM2, a production process manager for Node.js workloads to keep your app alive forever:

```bash
# Install PM2 globally
sudo npm install -g pm2

# Start the application
pm2 start server.js --name "tom-ai-backend"

# Ensure PM2 restarts automatically on VM reboot
pm2 startup
# (Run the sudo env command generated by pm2 startup, if prompt displays it)

# Save process list
pm2 save
```

To monitor backend logs or status, run:
```bash
pm2 status
pm2 logs tom-ai-backend
```

---

### Step 7: Update Frontend API Target
Now that your backend is running live on AWS:
1. Open the your local **frontend** directory project configurations: `frontend/.env`.
2. Locate the API URL configuration setting:
   ```properties
   REACT_APP_API_URL=http://<YOUR_EC2_PUBLIC_IP>:5000/api
   ```
3. Re-compile the frontend application:
   ```bash
   npm run build
   ```
4. Deploy the build chunk distribution files (`frontend/build/`) to your frontend hosting provider (like AWS S3 / CloudFront, Vercel, Netlify, or Amplify).
